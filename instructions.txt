# LifeCompass — Claude Code Technical Specification
*CTO handoff · April 2026*

---

## Tech Stack

| Layer | Choice | Rationale |
|---|---|---|
| Framework | React 18 (Vite scaffold) | Component model matches PRD's reusable input primitives |
| Styling | Inline styles only | PRD mandate — no CSS class injection, no Tailwind, no stylesheets |
| Charts | Custom SVG (no libraries) | PRD mandate — full control over bar grouping + color logic |
| State | useState + useMemo only | No external store needed; all state is single-session |
| Routing | None (step = integer in state) | 4 screens, no URL persistence required |
| Build | Vite + @vitejs/plugin-react | Fast HMR, standard JSX transform |
| Deploy | Static (Vercel / Netlify) | No backend, no auth, no DB — all computation is client-side |

## State Architecture (App.jsx)

Step 1 — Baseline:

    inc, exp, sav  →  each: { v: string, u: 'lakh' | 'crore' }
    age            →  string

Step 2 — Retirement:

    rTog   →  'do' | 'undecided' | null
    rAge   →  string
    rInc   →  { v: string, u: 'lakh' | 'crore' }

Step 2 — Children:

    kTog   →  'plan' | 'dont' | null
    nKid   →  1 | 2 | 3
    kYrs   →  string[]   // years until birth, one per child

Step 2 — Lifestyle Goals:

    goals  →  Array<{ id, type, cost:{v,u}, freq:'once'|'every', yrs:string }>

Navigation:

    step   →  0 | 1 | 2 | 3

Derived (useMemo):

    flows  →  deriveFlows({ inc, exp, sav, age, rTog, rAge, rInc, kTog, nKid, kYrs, goals })

## engine/deriveFlows.js

Assembles the flows[] array — the single source of truth for all results screens.

Flow shape:

    {
      id:           string,
      label:        string,       // e.g. "Child 1 — College"
      type:         'retirement' | 'education' | 'lifestyle',
      yearAway:     number,
      calendarYear: number,
      fv:           number,
      sip:          number,       // 0 if yearAway > SIP_CAP_YEARS
      emoji:        string
    }

Assembly rules:

- Retirement → exactly one flow (or zero if rTog !== 'do')
- Each child → one flow at yearsToBirth + 18, using EDU_INFLATION on BASE_COLLEGE_INR
- Lifestyle goals with freq === 'every' → expanded via expandRecurring()
- Flows with yearAway > SIP_CAP_YEARS → sip = 0, still appear in chart + outflow list
- totalSIP = sum of all flow.sip values

## Component Critical Rules (PRD §7)

1. All components defined at module level — never inside App or another component. React remounts on every render otherwise, destroying input focus.
2. No IIFEs in JSX — pre-compute all conditional JSX as named const variables above return.
3. Inline styles only — no className, no style tag injection.
4. MoneyBlank inputs store strings — parseFloat() only inside engine/ at calculation time.
5. Blank width — use hidden span mirror pattern: render typed value in invisible span, read offsetWidth, apply to input. Avoid ch units (unreliable with serif fonts).

## BarChart.jsx — SVG Logic

- Group flows[] by yearAway → chart columns
- Each column: stacked rects — retirement on top (orange #fb923c), education/lifestyle below (indigo #c7d2fe)
- Bars with total FV > ₹50L get a 4px dark cap (#ea580c retirement, #4f46e5 goals)
- Y-axis: 3 labels — 0, 50% of max, max (formatted with formatINR)
- X-axis: "Yr N" + "(YYYY)" in small sans-serif
- Wrap chart in overflowX: 'auto' div for mobile scroll
